## 一、准备工作

### 1.1 你需要准备的东西

| 物品 | 说明 | 获取方式 |
|------|------|----------|
| 阿里云服务器 | 已购买，知道公网 IP | 阿里云控制台 |
| 域名（可选） | 如 `example.com` | 阿里云域名控制台 |

### 1.2 确认服务器信息

打开阿里云控制台 → ECS 实例，记录以下信息：
- **公网 IP**：例如 `123.45.67.89`
- **系统**：CentOS 或 Ubuntu
- **用户名**：通常是 `root`
- **密码/密钥**：登录凭证

## 二、连接服务器

### 2.1 方式一：阿里云 Workbench（推荐小白）

1. 登录阿里云控制台
2. 进入 **ECS 实例列表**
3. 找到你的服务器，点击 **远程连接**
4. 点击 **Workbench 远程连接**
5. 输入密码登录

### 2.2 方式二：本地终端（Mac/Linux）

```bash
ssh root@123.45.67.89
# 输入密码（输入时不显示，输完回车）
```

### 2.3 方式三：Xshell/Putty（Windows）

1. 打开 Xshell
2. 新建会话，主机填服务器 IP
3. 用户名 `root`，输入密码
4. 连接

## 三、服务器环境安装

### 3.1 更新系统

```bash
# CentOS
sudo yum update -y

# Ubuntu
sudo apt-get update -y
```

### 3.2 安装 Node.js

```bash
# CentOS
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# Ubuntu
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装（看到版本号表示成功）
node -v
npm -v
```

### 3.3 安装 Nginx

```bash
# CentOS
sudo yum install -y nginx

# Ubuntu
sudo apt-get install -y nginx

# 启动 Nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# 验证（看到 active (running) 表示成功）
sudo systemctl status nginx
```

### 3.4 安装 PM2

```bash
# 全局安装 PM2
sudo npm install -g pm2

# 验证
pm2 -v
```

## 七、配置 Nginx

### 7.1 创建 Nginx 配置文件

```bash
# 创建配置文件
sudo vim /etc/nginx/conf.d/love-test.conf
```

**按 `i` 进入编辑模式，复制粘贴以下内容：**

```nginx
server {
    listen 80;
    server_name 你的域名或服务器 IP;  # 如果没有域名，填服务器 IP

    # 项目根目录
    set $project_root /data/test/love-test;

    # 访问日志
    access_log /var/log/nginx/love-test-access.log;
    error_log /var/log/nginx/love-test-error.log;

    # 首页 /love-test/
    location /love-test/ {
        alias $project_root/public/;
        index index.html;
        try_files $uri $uri/ /love-test/index.html;
    }

    # 测试页 /love-test/test
    location = /love-test/test {
        alias $project_root/public/test.html;
    }

    # 结果页 /love-test/result
    location = /love-test/result {
        alias $project_root/public/result.html;
    }

    # 静态资源
    location /love-test/src/ {
        alias $project_root/public/src/;
        expires 30d;
    }

    # API 接口
    location /love-test/api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # CORS
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
        add_header Access-Control-Allow-Headers 'Content-Type';

        if ($request_method = OPTIONS) {
            return 204;
        }
    }
}
```

**编辑完成后：**
- 按 `ESC` 键
- 输入 `:wq` 回车（保存并退出）

### 7.2 验证 Nginx 配置

```bash
# 验证配置是否正确（看到 successful 表示成功）
sudo nginx -t

# 重载 Nginx
sudo nginx -s reload
```

### 7.3 检查 Nginx 状态

```bash
sudo systemctl status nginx
# 看到 active (running) 表示成功
```

## 八、配置防火墙

### 8.1 CentOS (firewalld)

```bash
# 开放 HTTP 端口
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload

# 验证
sudo firewall-cmd --list-all
```

### 8.2 Ubuntu (ufw)

```bash
sudo ufw allow http
sudo ufw allow https
sudo ufw allow 8000/tcp
sudo ufw reload
sudo ufw status
```

### 8.3 阿里云安全组（重要！）

1. 登录阿里云控制台
2. 进入 **ECS 实例** → **安全组**
3. 点击 **配置规则** → **入方向**
4. 添加规则：
   - 端口：`80/80`，授权对象：`0.0.0.0/0`
   - 端口：`443/443`，授权对象：`0.0.0.0/0`
   - 端口：`8000/8000`，授权对象：`0.0.0.0/0`

## 十、配置域名（可选）

### 10.1 域名解析

1. 登录阿里云域名控制台
2. 找到你的域名，点击 **解析**
3. 添加记录：
   - 记录类型：`A`
   - 主机记录：`www`（或 `@`）
   - 记录值：你的服务器 IP
   - TTL：默认

### 10.2 修改 Nginx 配置

```bash
sudo vim /etc/nginx/conf.d/love-test.conf
```

修改 `server_name` 为你的域名：

```nginx
server_name example.com www.example.com;  # 改成你的域名
```

重载 Nginx：

```bash
sudo nginx -t && sudo nginx -s reload
```

### 10.3 访问测试

```
http://www.example.com/love-test/
```

## 十二、自动化部署（推荐）

### 12.1 配置服务器 IP

打开项目根目录的 `ecosystem.config.js` 文件，找到这一行：

```javascript
host: '你的服务器 IP',  // 修改为你的服务器 IP
```

改成你的服务器 IP，例如：

```javascript
host: '123.45.67.89',
```

保存并提交：

```bash
git add ecosystem.config.js
git commit -m "配置 PM2 部署 IP"
git push
```

### 12.2 首次部署（仅需一次）

在服务器上执行：

```bash
cd /data/test/love-test
pm2 deploy ecosystem.config.js production setup
```

这会：
1. 克隆代码到服务器
2. 安装依赖
3. 启动 PM2 服务

### 12.3 后续更新（每次代码修改后）

**在本地电脑上：**

```bash
# 1. 提交并推送代码
git add .
git commit -m "修改说明"
git push
```

**在服务器上：**

```bash
# 一行命令完成更新
pm2 deploy ecosystem.config.js production update
```

这会自动：
1. 拉取最新代码（git pull）
2. 安装依赖（npm install）
3. 重启服务（pm2 reload）

### 12.4 查看部署状态

```bash
# 查看部署历史
pm2 deploy ecosystem.config.js production status

# 查看服务状态
pm2 status
```
